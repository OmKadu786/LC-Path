01

Overview & architecture

LCPath is a Chrome extension that injects a side panel into every LeetCode problem page. It scrapes your solved problem list from your LeetCode profile, computes topic-level strength, and sends that context to the DeepSeek API to generate next-problem recommendations and answer chat questions.

🔍
Content script
Runs on leetcode.com pages. Scrapes your solved problems list and the current problem's tags. Sends data to the panel via chrome.runtime messages.
📋
Side panel UI
Two tabs: Home (recommendations + topic bars + stats) and Chat (free-form AI conversation with full LeetCode context pre-loaded).
🤖
DeepSeek API
Called from the panel when loading recommendations or sending a chat message. Your solved list is injected as system context on every call.
🗄️
chrome.storage
Caches your solved list for 6 hours so the extension doesn't re-scrape on every page load. Also stores your DeepSeek API key securely.

Data flow

01
Extension activates on leetcode.com
The content script fires on any leetcode.com/* URL. It checks chrome.storage for a cached solved list. If the cache is stale (>6h) or empty, it fetches leetcode.com/u/{username}/ and scrapes the solved table.
02
Scrape & parse solved problems
The content script reads each solved problem's title, difficulty, and topic tags from the DOM. It also reads the current problem's topic tags if you're on a problem page.
03
Side panel opens
The panel receives the solved list via message. It computes topic strength locally (count of solved problems per tag), then calls DeepSeek to get 3 next-problem recommendations.
04
Chat tab
Every chat message prepends your full solved context as a system prompt to DeepSeek: "The user has solved X problems. Solved: [list]. Current problem: [title + tags]." The AI answers as a personalized tutor.

content.js — data scraping

⚠️ LeetCode's DOM changes occasionally. If scraping breaks, inspect the profile page and update the selectors below. The selectors here are current as of 2025.
// content.js — runs on every leetcode.com page

const CACHE_KEY = 'lcpath_solved_cache';
const CACHE_TTL = 6 * 60 * 60 * 1000; // 6 hours in ms

// Get current problem's topic tags from the problem page
function getCurrentProblemData() {
  const title = document.querySelector('[data-cy="question-title"]')?.textContent?.trim();
  const tagEls = document.querySelectorAll('a.topic-tag__1jni');
  const tags = [...tagEls].map(el => el.textContent.trim());
  const diffEl = document.querySelector('[diff]') || document.querySelector('.css-10o4wqw');
  const difficulty = diffEl?.textContent?.trim() || 'Unknown';
  return { title, tags, difficulty };
}

// Fetch and parse the user's solved problem list from their profile
async function fetchSolvedProblems(username) {
  const url = `https://leetcode.com/u/${username}/`;
  const res = await fetch(url);
  const html = await res.text();
  const doc = new DOMParser().parseFromString(html, 'text/html');

  // Solved problems appear in a table on the profile page
  const rows = doc.querySelectorAll('.ac-solved-problems tr');
  const problems = [];

  rows.forEach(row => {
    const link = row.querySelector('a');
    const diffCell = row.querySelector('td:nth-child(2)');
    if (!link) return;
    problems.push({
      title: link.textContent.trim(),
      slug: link.href.split('/problems/')[1]?.replace('/', ''),
      difficulty: diffCell?.textContent?.trim() || 'Unknown'
    });
  });

  return problems;
}

// Main: get username from URL or storage, then gather all data
async function main() {
  const { username, lcpath_solved_cache, lcpath_cache_time } =
    await chrome.storage.local.get(['username', 'lcpath_solved_cache', 'lcpath_cache_time']);

  if (!username) return; // user hasn't set their username yet

  let solved = lcpath_solved_cache;
  const stale = !lcpath_cache_time || (Date.now() - lcpath_cache_time > CACHE_TTL);

  if (!solved || stale) {
    solved = await fetchSolvedProblems(username);
    await chrome.storage.local.set({
      lcpath_solved_cache: solved,
      lcpath_cache_time: Date.now()
    });
  }

  const currentProblem = getCurrentProblemData();

  // Send data to the side panel
  chrome.runtime.sendMessage({
    type: 'LCPATH_DATA',
    payload: { solved, currentProblem, username }
  });
}

main();
💡 Username setup: On first install, the panel shows a prompt asking for the user's LeetCode username. It's stored in chrome.storage.local and used to construct the profile URL for scraping.

panel.js — Home tab logic

// panel.js — home tab: stats, topic bars, recommendations

let userData = null;

// Listen for data from content.js
chrome.runtime.onMessage.addListener((msg) => {
  if (msg.type === 'LCPATH_DATA') {
    userData = msg.payload;
    renderHome(userData);
  }
});

// Render stats row
function renderStats(solved) {
  const easy = solved.filter(p => p.difficulty === 'Easy').length;
  const med  = solved.filter(p => p.difficulty === 'Medium').length;
  document.getElementById('stat-solved').textContent = solved.length;
  document.getElementById('stat-ratio').textContent =
    `${easy}E / ${med}M`;
}

// Compute topic strength from solved list
// Note: profile scrape doesn't include tags; we use the tag map below.
// For full tags, a secondary fetch of each problem page is needed.
// Simplified: use local TOPIC_MAP of known popular problems.
const TOPIC_MAP = {
  'two-sum': ['Array', 'Hash Table'],
  'valid-anagram': ['Hash Table', 'String'],
  'binary-search': ['Binary Search'],
  // ... extend this map or use LeetCode's GraphQL API (see note)
};

function computeTopicStrength(solved) {
  const counts = {};
  solved.forEach(p => {
    const tags = TOPIC_MAP[p.slug] || [];
    tags.forEach(tag => { counts[tag] = (counts[tag] || 0) + 1; });
  });
  const max = Math.max(...Object.values(counts), 1);
  return Object.entries(counts)
    .sort((a, b) => b[1] - a[1])
    .slice(0, 8)
    .map(([topic, count]) => ({ topic, pct: Math.round((count / max) * 100) }));
}

function renderTopicBars(solved) {
  const topics = computeTopicStrength(solved);
  const container = document.getElementById('topic-bars');
  container.innerHTML = topics.map(({ topic, pct }) => {
    const cls = pct > 65 ? 'strong' : pct > 35 ? 'medium' : 'weak';
    return `
      <div class="bar-row">
        <span class="bar-label">${topic}</span>
        <div class="bar-bg">
          <div class="bar-fill ${cls}" style="width:${pct}%"></div>
        </div>
        <span class="bar-pct">${pct}%</span>
      </div>`;
  }).join('');
}

// Ask DeepSeek for 3 recommendations — returns JSON
async function fetchRecommendations(solved, currentProblem, apiKey) {
  const solvedTitles = solved.map(p => p.title).join(', ');
  const prompt = `
You are a LeetCode study coach. The user has solved these problems: ${solvedTitles}.
They are currently looking at: ${currentProblem?.title || 'unknown'} (${currentProblem?.tags?.join(', ') || 'no tags'}).

Return exactly 3 next problem recommendations as JSON (no markdown, just the JSON):
[
  {
    "title": "Problem Name",
    "difficulty": "Easy|Medium|Hard",
    "topic": "main topic tag",
    "why": "one sentence reason based on their history"
  }
]
Focus on filling their weakest topic gaps while building on what they know.
  `;

  const res = await fetch('https://api.deepseek.com/v1/chat/completions', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${apiKey}`
    },
    body: JSON.stringify({
      model: 'deepseek-chat',
      messages: [{ role: 'user', content: prompt }],
      temperature: 0.7,
      max_tokens: 400
    })
  });

  const data = await res.json();
  const text = data.choices[0].message.content.trim();
  return JSON.parse(text);
}

function renderRecommendations(recs) {
  const container = document.getElementById('recommendations');
  container.innerHTML = recs.map((r, i) => `
    <div class="rec-card">
      <div class="rec-title">${r.title}</div>
      <div class="rec-meta">
        <span class="badge diff-${r.difficulty.toLowerCase()}">${r.difficulty}</span>
        <span class="topic-tag">${r.topic}</span>
        <button class="why-btn" data-why="${i}">why →</button>
      </div>
      <div class="why-box" id="why-${i}">${r.why}</div>
    </div>`
  ).join('');

  container.querySelectorAll('.why-btn').forEach(btn => {
    btn.addEventListener('click', () => {
      const box = document.getElementById(`why-${btn.dataset.why}`);
      box.classList.toggle('show');
    });
  });
}

async function renderHome({ solved, currentProblem }) {
  renderStats(solved);
  renderTopicBars(solved);
  const { deepseek_key } = await chrome.storage.local.get('deepseek_key');
  try {
    const recs = await fetchRecommendations(solved, currentProblem, deepseek_key);
    renderRecommendations(recs);
  } catch (e) {
    document.getElementById('recommendations').innerHTML =
      '<div class="error">Could not load recommendations. Check your API key.</div>';
  }
}

// Tab switching
document.querySelectorAll('.tab').forEach(tab => {
  tab.addEventListener('click', () => {
    document.querySelectorAll('.tab, .tab-content').forEach(el => el.classList.remove('active'));
    tab.classList.add('active');
    document.getElementById(`tab-${tab.dataset.tab}`).classList.add('active');
  });
});

chat.js — Chat tab + DeepSeek API

// chat.js — handles the Chat tab

const chatMessages = document.getElementById('chat-messages');
const chatInput   = document.getElementById('chat-input');
const chatSend    = document.getElementById('chat-send');

let chatHistory = []; // [{role, content}] — full conversation context

// Build the system prompt from the user's LeetCode data
function buildSystemPrompt(solved, currentProblem) {
  const solvedList = solved
    .map(p => `${p.title} (${p.difficulty})`)
    .join('\n');

  const current = currentProblem?.title
    ? `The user is currently working on: ${currentProblem.title} [${currentProblem.tags?.join(', ')}]`
    : 'The user is browsing LeetCode.';

  return `You are LCPath, a personalized LeetCode study coach.
You know exactly which problems this user has solved.

SOLVED PROBLEMS (${solved.length} total):
${solvedList}

${current}

Use this context to give highly personalized advice.
When recommending problems, always mention WHY based on their history.
Be concise — they're a developer, not a student. No fluff.`;
}

async function sendMessage() {
  const text = chatInput.value.trim();
  if (!text) return;
  chatInput.value = '';

  // Render user message
  addMessage('user', text);
  chatHistory.push({ role: 'user', content: text });

  // Thinking indicator
  const thinkingEl = addMessage('ai', '...');

  const { deepseek_key } = await chrome.storage.local.get('deepseek_key');

  try {
    const systemPrompt = buildSystemPrompt(
      userData?.solved || [],
      userData?.currentProblem
    );

    const res = await fetch('https://api.deepseek.com/v1/chat/completions', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${deepseek_key}`
      },
      body: JSON.stringify({
        model: 'deepseek-chat',
        messages: [
          { role: 'system', content: systemPrompt },
          ...chatHistory
        ],
        temperature: 0.8,
        max_tokens: 600
      })
    });

    const data = await res.json();
    const reply = data.choices[0].message.content;

    thinkingEl.textContent = reply;
    chatHistory.push({ role: 'assistant', content: reply });

  } catch (e) {
    thinkingEl.textContent = 'Error reaching DeepSeek. Check your API key in settings.';
    thinkingEl.classList.add('error');
  }

  chatMessages.scrollTop = chatMessages.scrollHeight;
}

function addMessage(role, text) {
  const el = document.createElement('div');
  el.className = `msg msg-${role}`;
  el.textContent = text;
  chatMessages.appendChild(el);
  chatMessages.scrollTop = chatMessages.scrollHeight;
  return el;
}

chatSend.addEventListener('click', sendMessage);
chatInput.addEventListener('keydown', (e) => {
  if (e.key === 'Enter' && !e.shiftKey) {
    e.preventDefault();
    sendMessage();
  }
});

DeepSeek API integration

LCPath uses deepseek-chat, DeepSeek's general-purpose model. The API is OpenAI-compatible — same endpoint structure, just a different base URL and key.

Getting your API key

1
Create an account
Go to platform.deepseek.com → sign up → verify email.
2
Generate an API key
Dashboard → API Keys → Create new key. Copy it — you won't see it again.
3
Top up credits
Add $2–5 — it will last months at typical usage. Billing → Add credits.
4
Paste into LCPath settings
Click the ⚙ icon in the panel → paste your key → save. It's stored locally in chrome.storage.local, never sent anywhere except DeepSeek's API.

Model choice

Model Use case Cost
deepseek-chat Recommendations + chat (what we use) ~$0.00014 / 1K tokens
deepseek-reasoner Hard algorithmic explanations (optional) ~$0.00055 / 1K tokens

A typical "give me 3 recommendations" call is ~800 tokens total. That's $0.0001 per call — less than 1 paisa.

Installation steps

Download / clone the repo
Put all the files in a folder called lcpath/ on your machine.
2
Open Chrome extensions
Navigate to chrome://extensions in your browser.
3
Enable developer mode
Toggle "Developer mode" in the top-right corner of the extensions page.

Load unpacked
Click "Load unpacked" → select your lcpath/ folder. The extension icon appears in your toolbar.

Set up LCPath

Open the side panel

Known risks & mitigations

⚠️ DOM scraping breaks when LeetCode updates their UI
LeetCode redesigns their DOM a few times a year. The selectors in content.js will need updating. Mitigation: keep the selectors in one place (top of content.js) so they're easy to patch. Consider using LeetCode's unofficial GraphQL API as a more stable alternative — it's what third-party tools like LeetCode Stats use.
⚠️ Topic tags not available from profile scrape alone
The profile page lists solved problems but not their topic tags. Mitigation (simple): maintain a local TOPIC_MAP of the 300 most common problems and their tags. Mitigation (robust): fetch each problem's tags via LeetCode's GraphQL endpoint: POST https://leetcode.com/graphql with the questionData query. Rate-limit to 1 req/sec.
⚠️ DeepSeek sometimes returns malformed JSON for recommendations
The model occasionally wraps JSON in markdown backticks. Mitigation: strip ```json fences before JSON.parse(), and wrap in a try/catch with a fallback message.
⚠️ API key stored in chrome.storage.local
chrome.storage.local is not encrypted but is sandboxed to the extension. It's as safe as storing a password in a password manager extension. Mitigation: never log the key, and remind users to use a key with spending limits set on DeepSeek's dashboard.
ℹ️ LeetCode may block profile page fetches
Fetching someone's public profile page is not against LeetCode's ToS (it's public data). Rate limiting to once per 6 hours makes this a non-issue in practice.
